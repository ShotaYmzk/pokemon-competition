# デッキ構築ルール

## 制約（絶対）
- デッキ合計: 60枚ちょうど
- 同名カード: 4枚以下（基本エネルギーは例外・枚数制限なし）
- 使用可能カード: **指定カードリスト内のカードIDのみ（約1,267種、ID 1–1,267）**
  - CSV の 2,022行はワザ行を含む行数であり、プールサイズではない

## デッキ設計の進め方

- AI エージェントにデッキ構築させる必要はない（FAQ）。`deck.csv` を人間が設計して固定 submit 可
- 「約1,267種から60枚を選ぶ」問題は、MCTS 開発と並行してオフラインで最適化してよい
- 試作は `agent/decks/legacy/YYYY-MM-DD_<concept>.csv`、確定は `agent/deck.csv`

## 基本エネルギーカードID（枚数制限なし）
- 1: 基本【草】エネルギー
- 2: 基本【炎】エネルギー
- 3: 基本【水】エネルギー
- 4: 基本【雷】エネルギー
- 5: 基本【超】エネルギー
- 6: 基本【闘】エネルギー
- 7: 基本【悪】エネルギー
- 8: 基本【鋼】エネルギー

## デッキJSON形式
{
  "deck": [
    {"card_id": 121, "count": 2},
    ...
  ]
}

## 提出形式（Kaggle）
- `agent/deck.csv`: カードIDを1行1枚、60行、ヘッダーなし
- 例：カードID 121を2枚入れるなら「121」と書かれた行が2行
