# コンペティションルール（厳守）

## カードプール

- FAQ: 「本コンテストで指定された特定のカードリストにあるカードのみが使用可能」
- 指定リスト: Kaggle Strategy ページの Data セクション（CSV + PDF）
- ローカル参照:
  - `pokemon-tcg-ai-battle/JP_Card_Data.csv` — 2,022行（**ワザ行を含む。1行≠1カード**）
  - `Card_ID List_JP.pdf` — カードID 1–1,267
- **ユニークカード数: 約1,267種**（`AllCard()` / `all_card_data()` の結果と一致）
- デッキ・実装で使う ID は **1–1,267 の指定集合内** に限定

## デッキ制約

- 60枚ちょうど（標準ポケカフォーマット）
- 同名カード最大4枚（基本エネルギー ID 1–8 は枚数制限なし）
- **デッキ構築を AI に任せる必要はない** — 人間が `deck.csv` を決め打ちして submit 可。エージェントは対戦中の手番選択のみ担当すればよい
- デッキ最適化は MCTS / エージェント開発と**独立した作業**として進めてよい

## やってはいけないこと
- 指定カードリスト（ID 1–1,267 外）のカードIDを使う
- カードの効果・ダメージ値をコード内にハードコードする（CSVから動的に読む）
- デッキを60枚以外の枚数で構築する
- 同名カードを5枚以上デッキに入れる
- 意思決定ループにタイムアウト処理を忘れる（上限: 10分/試合）
- 主催者提供以外のゲームシミュレーターを使う

## ストラテジー部門の評価軸（実装方針に反映すること）
1. エージェントの安定性（クラッシュしない・時間切れしない）
2. デッキのコンセプト設計（明確な戦略意図）
3. シミュレーション部門での成績

## 提出制限
- Simulation: 1日5回まで → 不用意な提出スクリプト実行に注意
- Strategy: 1チーム1回のみ → レポートは慎重に最終化してから提出
